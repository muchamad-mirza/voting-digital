\# Digital Voting System (VotingDigital)



A desktop-based digital voting application built to facilitate secure and transparent student elections. This system ensures voting integrity by implementing custom session management and a strict "one student, one vote" validation mechanism.



\## 📖 Project Overview

This project was developed as a final academic assignment during our first year of undergraduate studies. It serves as a foundational milestone in our software engineering journey, marking our first comprehensive project using Java. 



The application was designed to simulate a real-world voting environment with built-in safeguards, including local data tracking for student credentials (NIS) and real-time vote tabulation.



\## ✨ Features

\* \*\*Custom Authentication:\*\* A secure login mechanism utilizing a Student ID (NIS) and password validation.

\* \*\*Session Management:\*\* Tracks the active user to prevent unauthorized access to voting modules.

\* \*\*Voting Integrity Protocol:\*\* Enforces a strict one-time voting rule per registered student, storing vote states locally.

\* \*\*Real-time Statistics:\*\* A dashboard that dynamically calculates and displays the current vote counts for each candidate.

\* \*\*Intuitive UI/UX:\*\* A straightforward GUI built from scratch to ensure an accessible user experience.



\## 🛠️ Tech Stack

\* \*\*Language:\*\* Java (Native)

\* \*\*GUI Framework:\*\* Java Swing / AWT

\* \*\*IDE:\*\* Apache NetBeans

\* \*\*Storage:\*\* Local data structures / File-based handling

\## 📝 Developer's Note: The Journey

“Crafted by hand, driven by curiosity.”



As or very first programming project, Java was the first language we ever learned. At the time of development, we had not yet been formally introduced to relational databases (SQL), computer networking, or advanced information security. However, we was determined to make the system as secure and functional as possible.



We engineered our own login logic, session handling, and local storage mechanisms to prevent double-voting. Every line of backend logic and frontend design was written manually, without the assistance of AI code generators. The goal was to build raw problem-solving skills.
